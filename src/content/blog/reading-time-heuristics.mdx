Reading time is useful on index cards, but raw word counts lie when a post contains fenced code, display math, or custom MDX blocks that take longer to scan than plain text.

This site computes reading time at **build time** by stripping non-prose regions before counting words:

- YAML frontmatter
- Fenced code blocks
- Inline and display math
- MDX component tags

You can still override the estimate in frontmatter when a post is mostly visual:

```yaml
readingTimeMinutes: 8
```

## A tiny example

If a paragraph mentions $O(n \log n)$ complexity inline, the math tokens are excluded from the word count so the estimate stays closer to actual reading effort for prose.

Display formulas get the same treatment:

$$
T(n) = \begin{cases}
1 & n \le 1 \\
2T(n/2) + n & n > 1
\end{cases}
$$

The goal is not perfection on v1 — it is a **consistent baseline** that can gain per-component weights later without changing the post format.
